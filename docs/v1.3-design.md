# Deck v1.3 — Design

Session chains and model comparison.

## Pillars

### Pillar 1: Session Chains
Link related sessions together to track multi-session workflows.

- New concept: a "chain" is a named group of sessions working toward a goal
- `/chains` page showing all chains with their sessions
- Create a chain from the session detail page ("Add to chain" button)
- Chain card: name, sessions count, total cost, total duration, date range
- Click chain → expanded view with all sessions in order, total stats, progress visualization
- Chains stored in `~/.deck/chains.json`

**Files:** `src/app/chains/page.tsx` (new), `src/app/api/chains/route.ts` (new), `src/lib/chains.ts` (new), update sidebar, update command palette.

### Pillar 2: Model Comparison
A/B analysis comparing model performance across sessions.

- `/models` page dedicated to model analysis
- Head-to-head comparison table: pick two models, compare avg cost, avg tokens, avg duration, cache efficiency
- Model usage trend: stacked area chart showing which models are used over time
- Model efficiency scatter plot: cost vs speed for each model (bubble size = session count)
- Recommendation: "Based on your usage, switching X% of [model] sessions to [model] could save $Y"

**Files:** `src/app/models/page.tsx` (new), `src/app/api/models/route.ts` (new), update sidebar, update command palette.

### Pillar 3: Responsive Polish
Make all pages work well on smaller screens.

- Audit and fix responsive breakpoints across all pages
- Ensure sidebar collapses on mobile (auto-collapse below lg breakpoint)
- Stack grids to single column on small screens
- Ensure charts resize properly
- Add a mobile-friendly bottom nav bar (shows on sm only, hides on lg+)

**Files:** `src/components/mobile-nav.tsx` (new), update `src/components/sidebar.tsx`, update `src/app/layout.tsx`.
