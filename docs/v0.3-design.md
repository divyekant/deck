# Deck v0.3 — Design Document

## Overview

v0.3 focuses on three pillars: hardening the foundation, polishing UX, and adding high-value features.

## Pillar 1: Foundation Hardening

### 1.1 Pagination
- `GET /api/sessions` accepts `?page=1&limit=50` params
- Returns `{ sessions, total, page, totalPages }`
- Sessions list page shows pagination controls (prev/next/page numbers)
- Default: 50 sessions per page

### 1.2 Type Safety
- Replace all `any` types in `message-view.tsx` with `ContentBlock` union
- Replace all `any` types in `sessions/[id]/page.tsx` with proper typed interfaces
- Remove all `eslint-disable` comments for `@typescript-eslint/no-explicit-any`
- `StreamMessage` interface should properly type `message` field

### 1.3 Deduplicate Process Management
- Extract shared logic from `startSession` and `resumeSession` into a `spawnClaudeProcess` helper
- Single place for: spawn, stdout/stderr buffering, listener management, cleanup timer
- `startSession` and `resumeSession` become thin wrappers that build args and call the helper

### 1.4 API Input Validation
- `/api/sessions/start`: validate projectDir exists (fs.access), model is in known list, prompt is non-empty
- `/api/sessions/resume`: validate sessionId format (UUID), prompt non-empty
- `/api/mcp` POST: validate server name non-empty, command non-empty
- Return 400 with descriptive error for validation failures, 500 for internal errors

## Pillar 2: UX Polish

### 2.1 Command Palette (Cmd+K)
- Global keyboard listener for Cmd+K (or Ctrl+K)
- Dialog with search input, shows:
  - Pages: Home, Sessions, Repos, MCP, Settings
  - Recent sessions (top 5, searchable by prompt text)
  - Actions: New Session
- Arrow keys + Enter to navigate, Escape to close
- New component: `src/components/command-palette.tsx`
- Mounted in `layout.tsx`

### 2.2 Dark/Light Theme Toggle
- Use CSS variables for theme colors (zinc palette for dark, slate/neutral for light)
- `ThemeProvider` component wrapping app in layout.tsx
- Toggle button in sidebar footer (sun/moon icon)
- Reads/writes `settings.theme` via API
- Apply `dark`/`light` class on `<html>` element
- All component classes already use Tailwind dark: prefix convention

### 2.3 Session List Auto-Refresh
- Refetch sessions on window focus (visibilitychange event)
- Also poll every 30 seconds when tab is visible
- Show subtle "Updated" indicator when new data arrives

### 2.4 Keyboard Shortcuts
- `j`/`k` for up/down in sessions table (highlight row)
- `Enter` to open highlighted session
- `Escape` to go back / close dialogs
- `/` to focus search in sessions list
- Show shortcut hints in command palette

## Pillar 3: New Features

### 3.1 Cost Trend Chart
- New component: `src/components/cost-trend-chart.tsx`
- Line chart (CSS-based, no external library) showing daily cost
- Toggle between 7/30/90 day views
- Placed on overview page, replaces or sits alongside activity chart
- Data: extend `getOverviewStats` to accept `days` param, or add `getCostTrend(days)` function

### 3.2 Session Bookmarks
- Star icon on session rows and session detail header
- Bookmarks stored in `~/.deck/bookmarks.json` (array of session IDs)
- New API: `GET /api/bookmarks`, `POST /api/bookmarks` (toggle)
- New lib: `src/lib/bookmarks.ts` (getBookmarks, toggleBookmark)
- Filter option on sessions list: "Bookmarked only"
- Bookmarked sessions highlighted with a star icon

### 3.3 Export Session to Markdown
- Button on session detail page header: "Export"
- Generates markdown with:
  - Session metadata header (project, model, date, cost, tokens)
  - Full conversation (user prompts, assistant responses)
  - Tool calls as code blocks
  - Thinking blocks as collapsible sections (using `<details>`)
- Downloads as `session-<id-prefix>.md`
- Utility: `src/lib/claude/export.ts`

### 3.4 Budget Projection
- In BudgetWidget, show "Projected: $X" based on daily burn rate
- Calculate: (cost so far this period / days elapsed) * days in month
- Show as dotted line or secondary text below progress bar
- Warning color when projection exceeds budget

### 3.5 Date Range Filter
- Add date range controls to sessions list filters
- Quick presets: Today, Last 7 days, Last 30 days, All time
- Custom date range picker (simple from/to inputs)
- Filter applied client-side (sessions already loaded with pagination)

## Non-Goals (v0.3)
- No external chart libraries (keep CSS-based)
- No database (keep filesystem-only)
- No multi-user support
- No session deletion (read-only access to JSONL files)
- No mobile-specific layout (responsive but desktop-first)
