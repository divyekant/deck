# Deck v0.9 — Design

Last reference item (Ports) + two high-value features.

## Pillars

### Pillar 1: Ports — MCP Server Monitor
Show active MCP servers and their connection status.

- `/ports` page showing all configured MCP servers from Claude's config
- Read `~/.claude/settings.json` and project-level `.mcp.json` files for MCP server configs
- For each server: name, type (stdio/sse), command/url, status indicator
- Group by: Global vs Project-scoped
- Show which projects reference each server

**Files:** `src/app/ports/page.tsx` (new), `src/app/api/ports/route.ts` (new), update sidebar, update command palette.

### Pillar 2: Session Comparison
Side-by-side comparison of two sessions.

- `/compare` page with two session pickers
- Compare: cost, tokens, duration, message count, model, files modified
- Visual diff bars showing relative values
- Highlight which session was more efficient (cost per message, tokens per message)

**Files:** `src/app/compare/page.tsx` (new), update command palette.

### Pillar 3: Context Window Visualization
Visualize token usage within a session — how the context window fills up.

- New component on session detail page: context window chart
- Stacked area chart (SVG) showing cumulative input/output/cache tokens per message
- Visual indicator of context window capacity (200K for most models)
- Shows where the conversation hit peak token usage

**Files:** `src/components/context-window-chart.tsx` (new), update `src/app/sessions/[id]/page.tsx`.
