# Deck v0.1 — Design Document

## What
A local web dashboard that wraps Claude Code CLI. You interact through the dashboard UI; CC CLI runs underneath.

## Architecture

```
Browser (Next.js frontend)
    ↕ HTTP / SSE
Next.js API routes (orchestration layer)
    ↕ spawn / stdio
Claude Code CLI (engine)
    + reads ~/.claude/ for historical data
```

## Data Sources

### Session JSONL files
- Location: `~/.claude/projects/<escaped-path>/<session-uuid>.jsonl`
- Each line is JSON with a `type` field: `user`, `assistant`, `progress`, `queue-operation`, `file-history-snapshot`
- Assistant messages contain: `message.model`, `message.usage` (input_tokens, output_tokens, cache_creation_input_tokens, cache_read_input_tokens), `message.content` (array of thinking/text/tool_use blocks), `message.stop_reason`
- User messages contain: `message.role`, `message.content` (string or tool_result array)
- All messages have: `sessionId`, `timestamp`, `uuid`, `parentUuid`

### History file
- Location: `~/.claude/history.jsonl`
- Each line: `{ sessionId, project, timestamp, display (prompt text) }`

### Model pricing (hardcoded for v0.1)
| Model | Input $/M | Output $/M | Cache Write $/M | Cache Read $/M |
|-------|-----------|------------|-----------------|----------------|
| claude-opus-4-6 | 15.00 | 75.00 | 18.75 | 1.50 |
| claude-sonnet-4-6 | 3.00 | 15.00 | 3.75 | 0.30 |

## Pages

### 1. Overview (`/`)
The main dashboard. Shows:
- Greeting line (time-aware: "Good morning", etc.)
- Summary stats cards: Total Repos, Commits Today, Total Sessions, Estimated Cost
- Budget progress bar (hardcoded $500 default for v0.1)
- Activity chart (sessions per day, last 30 days) — use a simple bar chart with divs, no chart library
- Cost by Model breakdown (list with colored dots)
- Recent Sessions list (last 5, with project name, first prompt, timestamp, model)

### 2. Sessions (`/sessions`)
- Table of all sessions across all projects
- Columns: Project, First Prompt, Model, Messages, Tokens, Cost, Date
- Click row → session detail
- Sort by date (newest first)

### 3. Session Detail (`/sessions/[id]`)
- Header: project, model, date, duration, total cost
- Conversation view: alternating user/assistant messages
- Tool calls shown as collapsible blocks
- Token usage sidebar or summary at top
- Thinking blocks shown collapsed by default

### 4. New Session (`/sessions/new`)
- Select project directory (dropdown of known repos from history)
- Model selector (sonnet default, opus, haiku)
- Prompt textarea
- "Launch" button
- On launch: spawns `claude -p --output-format=stream-json --include-partial-messages --model <model> --session-id <uuid>`
- Streams output via SSE to the browser
- Shows live conversation as it happens

## API Routes

### `GET /api/sessions`
Parse all JSONL files from `~/.claude/projects/`. Return list of sessions with metadata.
Cache in memory with file mtime checks.

### `GET /api/sessions/[id]`
Parse specific session JSONL. Return full conversation with computed cost.

### `GET /api/stats`
Aggregate stats: session count, total cost, model breakdown, daily activity.

### `POST /api/sessions/start`
Body: `{ projectDir, model, prompt }`
Spawns `claude -p --output-format=stream-json --include-partial-messages --model <model> --session-id <generated-uuid>` in the projectDir.
Returns `{ sessionId }` and begins streaming.

### `GET /api/sessions/[id]/stream`
SSE endpoint. Pipes stdout from running claude process to browser.

### `POST /api/sessions/[id]/input`
Send additional input to a running session's stdin (for follow-up prompts).

## Tech Stack
- Next.js 15 (App Router)
- TypeScript
- Tailwind CSS v4
- shadcn/ui (zinc dark theme)
- No database — all reads from filesystem
- No chart library — CSS/div-based charts

## Theme
Dark theme matching the inspiration screenshot. Zinc-based colors from shadcn.
Sidebar navigation on the left. Main content area on the right.

## File Structure
```
src/
  app/
    layout.tsx          — root layout with sidebar
    page.tsx            — overview dashboard
    sessions/
      page.tsx          — session list
      [id]/
        page.tsx        — session detail
      new/
        page.tsx        — new session launcher
    api/
      sessions/
        route.ts        — GET list sessions
        [id]/
          route.ts      — GET session detail
          stream/
            route.ts    — SSE stream
          input/
            route.ts    — POST send input
        start/
          route.ts      — POST start session
      stats/
        route.ts        — GET aggregate stats
  lib/
    claude/
      parser.ts         — JSONL parsing utilities
      sessions.ts       — session discovery and listing
      costs.ts          — cost calculation from token usage
      process.ts        — spawn and manage claude CLI processes
    utils.ts            — shadcn utility
  components/
    ui/                 — shadcn components
    sidebar.tsx         — navigation sidebar
    session-card.tsx    — session preview card
    message-view.tsx    — render a conversation message
    tool-call-view.tsx  — render a tool call block
    stats-card.tsx      — metric card with label and value
    activity-chart.tsx  — simple div-based bar chart
    cost-breakdown.tsx  — model cost list
```
