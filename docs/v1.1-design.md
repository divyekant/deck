# Deck v1.1 — Design

Deep token analytics and cost optimization insights.

## Pillars

### Pillar 1: Token Analytics Page
Dedicated page for deep token usage analysis.

- `/tokens` page with:
  - **Cache efficiency chart**: cache hit rate over time (line chart, last 30 sessions)
  - **Token distribution**: pie/donut chart showing input vs output vs cache-read vs cache-write proportions
  - **Top token consumers**: table of sessions sorted by total tokens, with cost-per-token metric
  - **Model token comparison**: average tokens per session by model
  - **Daily token trend**: bar chart of tokens consumed per day (last 30 days)

**Files:** `src/app/tokens/page.tsx` (new), `src/app/api/tokens/route.ts` (new), update sidebar, update command palette.

### Pillar 2: Cost Optimization Tips
Smart tips based on usage patterns.

- New component on the Costs page: "Optimization Insights" card
- Analyze session data to generate tips:
  - "You could save $X by using Haiku for short sessions (<10 messages)"
  - "Your cache hit rate is X% — sessions with longer context benefit most from caching"
  - "Your most expensive hour is Xpm — consider batching work"
  - "X% of your cost comes from a single project"
- Each tip has an icon, text, and estimated savings

**Files:** `src/components/cost-tips.tsx` (new), update `src/app/costs/page.tsx`.

### Pillar 3: Session Heatmap
GitHub-style contribution heatmap on the analytics page.

- Full-year heatmap (52 weeks x 7 days grid) showing session activity
- Color intensity based on session count per day
- Tooltip on hover showing date and count
- Replace or supplement the existing peak usage hours chart

**Files:** `src/components/session-heatmap.tsx` (new), update `src/app/analytics/page.tsx`.
